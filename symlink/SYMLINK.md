คำสั่ง `ln -s cloud target` ในระบบ Unix/Linux ใช้สร้าง **Symbolic Link** (หรือเรียกว่า **Symlink** หรือ **Soft Link**) ชื่อ `target` ที่ชี้ไปยังไฟล์หรือโฟลเดอร์ `cloud`

---

### **Symbolic Link คืออะไร?**

Symbolic Link คือไฟล์พิเศษที่ทำหน้าที่เป็น **ทางลัด** (shortcut) ชี้ไปยังไฟล์หรือโฟลเดอร์อื่น ๆ ในระบบ คล้ายกับ Shortcut ใน Windows แต่สามารถชี้ไปยังไฟล์หรือโฟลเดอร์ที่อยู่ในไดรฟ์หรือระบบไฟล์ต่างกันได้ และสามารถชี้ไปยังไฟล์ที่ยังไม่มีอยู่ในขณะนั้นก็ได้

---

### **อธิบายคำสั่ง**

- **`ln`**: คำสั่งสำหรับสร้างลิงก์
- **`-s`**: ระบุว่าต้องการสร้าง **Symbolic Link** (Soft Link)
- **`cloud`**: **ไฟล์หรือโฟลเดอร์ต้นทาง** (ที่ต้องการชี้ไป)
- **`target`**: **ชื่อของ Symbolic Link** (ทางลัดที่สร้างขึ้น)

---

### **ตัวอย่างการใช้งาน**

หากคุณรัน:

```bash
ln -s /path/to/cloud /path/to/target
```

- จะสร้าง Symbolic Link ชื่อ `target` ใน `/path/to/`
- เมื่อคุณเข้าถึง `/path/to/target` ระบบจะนำคุณไปยัง `/path/to/cloud` แทน

---

### **ข้อควรรู้**

1. **Symbolic Link สามารถชี้ไปยังไฟล์หรือโฟลเดอร์ก็ได้**
2. **หากไฟล์ต้นทาง (`cloud`) ถูกลบ Symbolic Link (`target`) จะกลายเป็นลิงก์ที่หัก (broken link)**
3. **Symbolic Link สามารถชี้ข้ามระบบไฟล์ (filesystem) ได้**
4. **ใช้ `ls -l` เพื่อดูว่าลิงก์ชี้ไปที่ไหน:**
   ```bash
   ls -l /path/to/target
   ```
   ผลลัพธ์จะแสดงแบบนี้:
   ```
   lrwxrwxrwx 1 user group 11 Jul 13 12:00 target -> cloud
   ```
   (`->` จะแสดงว่าลิงก์ชี้ไปที่ไหน)

---

### **การใช้งานทั่วไป**

- สร้างทางลัดไปยังไฟล์หรือโฟลเดอร์ที่ใช้บ่อย
- จัดระเบียบไฟล์โดยไม่ต้องคัดลอก
- จัดการไฟล์หรือโฟลเดอร์หลายเวอร์ชัน
